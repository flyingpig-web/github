# bohun-AR1 개발 문서

> **콘텐츠명:** 한국광복군 대원의 긴급 임무 (AR.1 한국광복군 OSS 훈련)
> **원본 기획:** `pdf/AR1_한국광복군_프로토타입.pdf` (48p)
> **상태:** 기획 분석 완료 / 세부 기획 대기 중 (자산·수치 확정 후 착수)

이 문서는 PDF 프로토타입을 분석해 정리한 **개발 착수용 정리본**이다. 세부 기획이 나오면
"열린 항목(Open Items)" 섹션을 채워가며 진행한다.

---

## 1. 기술 스택 / 원칙

- **jQuery 3.6 + jQuery UI + 순수 HTML/CSS/JS** (기존 `lowerEl-*`, `preSchool-*` 와 동일 컨벤션)
- **non-SPA**: 화면(스텝)별 `.html` 파일 분리. 저사양 기기 타깃이라 SPA 사용 안 함.
- 게임 2종(EXP②/③)만 `<canvas>` 2D 사용. 3D/WebGL/물리엔진 불필요.
- **화면 규격: 953 × 537** (프로젝트 공통, README 기준)
- 외부 의존성 최소화. 수료증 캡처용 라이브러리(html2canvas 등)만 검토.

### 저사양 대응 가이드
- 게임 루프는 `requestAnimationFrame`, 운전 게임은 30fps 타깃 허용.
- 파티클(흙먼지) 개수 상한 설정.
- 맵/배경은 단일 이미지. 스프라이트 회전·카메라 추적만 Canvas로 처리.
- 이미지 프리로드 후 게임 시작.

## 2. 전체 플로우

```
0_TITLE → 1_INTRO(도입) → [2_EXP①-AR] → 3_Bridge① → 4_EXP②(모스) → 5_Bridge② → 6_EXP③(운전) → 7_END
   HTML       HTML          유니티(제외)      HTML        HTML(게임)       HTML         HTML(게임)      HTML
```

- **[2_EXP①-AR]은 유니티 담당(다른 개발자) → 웹 구현 범위 제외.**
  - 유니티 진입 직전(`목표①` 확인) / 복귀 직후(`3_Bridge①`) 연결 지점만 협의 필요.
- 각 스텝은 "터치하여 시작 / 스킵 / 다음" 으로 진행. 도입·연결은 이미지전환+자막 중심.

## 4. ★게임1 — EXP② 모스부호 리듬게임

**컨셉:** 모스 부호에 맞춰 전신기를 터치해 본부와 교신.

### 화면 구성
- 배경: 나무 책상 위 모스 전신기.
- 상단 우측: [설정] [튜토리얼] [홈] 버튼.
- 좌측: **성공률 표시 (n/20)**.
- 미션 창: 우→좌로 흐르는 모스 신호 + **판정라인**.
- 하단: 모스 전신기(터치 대상).

### 게임 규칙
1. 화면 아무 곳이나 터치 → PLAY 시작, 미션 창 팝업.
2. 신호가 **오른쪽 → 왼쪽**으로 흘러가며 판정라인으로 접근.
3. 판정라인에 닿는 순간 **모스 전신기 터치**.
4. **±5% 영역** 내 타이밍 → `Good!`(사운드 + 판정라인 네온 효과), 어긋나면 `Bad!`(사운드).
5. **총 20개 신호 중 10개 이상 성공 → 교신 완료**(체험 종료 팝업).

### ✅ 확정 결정
- **점(·)/선(–) 구분 없이 "탭 타이밍"만으로 판정** (프로토타입 단순화). hold(길게누르기) 판정 미적용.

### 구현 메모
- 1레인 리듬게임. 신호 위치 = `(판정시각 - 현재시각) × 속도` 시간기반 계산.
- 신호 데이터: 예시 `. .- --. .-.. .` = EAGLE / `... . -. -..` = SEND. → 실제 출제 시퀀스는 기획 확정 필요.
- Canvas 한 장 권장(신호=단순 도형). 가벼움.

### 팝업
- **설정:** 게임 일시정지 + 딤드. 배경음 ON/OFF, 효과음 ON/OFF. [X] → 복귀.
- **튜토리얼:** 일시정지 + 딤드. 게임 방법 3단계 안내. [X] → 복귀.
  - ① 흘러나오는 신호의 길이를 확인 ② 판정선에 닿는 순간 길이에 맞게 전신기 누름 ③ 20개 중 10개↑ 성공.
- **홈:** 0_TITLE 이동.
- **체험 종료:** "총사령부 교신 완료!" + [다음으로](→5_Bridge②) / [다시 하기](→메인).

### 사운드
- (e) 모스부호 사운드, (e) Good!, (e) Bad!

## 5. ★게임2 — EXP③ 탑다운 짚차 운전게임

**컨셉:** 지프를 운전해 총사령부에서 서신을 받아 3곳 부대(지대)에 전달.

### 화면 구성 (탑다운 맵)
- 맵 요소: 산악지형, 이동 경로(이동 방향 표시), 각 막사.
  - 거점: **충칭 한국광복군 총사령부(태극기)** / 2지대 / 진화 1지대 2구대 / 푸양 3지대 / 라오허카우 1지대 1구대.
- UI: 좌하단 **조이스틱**, 상단 좌측 **서신 전달 카운트(0/3)**, 우상단 [설정][튜토리얼][홈].
- 에셋: 서신(아이콘) 3개, 태극기, 막사.

### 게임 규칙
1. 터치 → PLAY 시작.
2. **조이스틱으로 차량 전진/후진 + 방향 조절.**
3. **경로 이탈(±10%) 시** → 흙먼지 파티클 이펙트 + 차량 전진 속도 50% 감소.
4. **총사령부(태극기) 통과** → `서신 0/0 → 3/3`, 서신 3개가 차량에 실리는 연출.
5. **각 지대 막사(3곳) 체크포인트 통과** → `3/3 → 2/3 → 1/3 → 0/3`, 서신이 해당 막사에 놓임.
6. **서신 3개 모두 전달 → 게임 종료**(체험 종료 팝업).

### ✅ 확정 결정
- 가상 조이스틱은 **새 이미지 소스 사용**(전달 예정).

### 구현 메모
- 2D 탑다운 + Canvas. 차량 스프라이트 회전 + 카메라 추적 + 파티클.
- 경로 이탈 판정 = 경로 폴리라인과 차량 위치의 거리 계산(임계 ±10%).
- 체크포인트 = 각 막사/태극기 위치 반경 판정.

### 팝업
- **설정 / 튜토리얼 / 홈:** 게임1과 동일 패턴.
  - 튜토리얼: ① 조이스틱으로 지프 운전 ② 경로 이탈 시 속도 느려짐 ③ 체크포인트 통과 시 서신 자동 전달.
- **체험 종료:** "서신 전달 완료!" + [다음으로](→7_END) / [다시 하기](→메인).

## 6. 7_END — 수료증

### 구성
- 배경: 제2지대 막사 딤드 화면.
- 좌측: [이름 작성] input + 임명 메시지("○○○ 을/를 한국광복군 제2지대 대원으로 임명합니다").
- 우측: **수료증** (한국광복군 제2지대 OSS 수료증).
  - 소속/이름(자동완성)/문구/직인(이범석). 수료증 번호 = **YYMMDD 자동 생성**.
- 버튼: [수료증 스크랩] / [처음으로](→0_TITLE).

### ✅ 확정 결정 — 수료증 저장 방식
- 수료증은 **이미지 1장**으로 깔고, 그 위에 **`position:absolute` 로 input(이름 등) 요소 배치**.
- [수료증 스크랩] = 수료증 영역을 **이미지로 캡처**(html2canvas류 캡처 개념)하여 다운로드.
- 번호 자동 생성: `YYMMDD`. 이름은 input → 수료증 표시 영역에 동기화.

## 7. 제안 파일 구조 (기존 컨벤션 기준)

```
bohun-AR1/
├─ index.html          # 0_TITLE
├─ intro.html          # 1_INTRO (도입 시퀀스 + 목표①)
├─ bridge1.html        # 3_Bridge① (연결① + 목표②)   ※유니티 복귀 진입점
├─ exp2.html           # 4_EXP②  모스부호 게임
├─ bridge2.html        # 5_Bridge② (연결② + 목표③)
├─ exp3.html           # 6_EXP③  짚차 운전 게임
├─ end.html            # 7_END   수료증
├─ css/
├─ js/
│   ├─ common.js       # 공통(전환/사운드/팝업 등)
│   ├─ intro.js
│   ├─ bridge1.js / bridge2.js
│   ├─ exp2.js         # 모스 게임 로직
│   ├─ exp3.js         # 운전 게임 로직
│   └─ end.js          # 수료증/캡처
├─ img/                # 배경, UI, 에셋(서신/태극기/막사/조이스틱 등)
├─ sound/              # 내레이션, 효과음(모스/Good/Bad/배경음)
└─ pdf/                # 원본 기획
```

> 스텝 간 이동은 페이지 네비게이션(`location.href`) 또는 기존 프로젝트 전환 방식 따름.
> 화면 단위 분리이므로 스텝 추가/수정이 독립적.

## 8. 열린 항목 (Open Items — 세부 기획 확정 시 채울 것)

- [ ] **자산 일체 수급**: 배경/UI/에셋 이미지, 사운드(내레이션·효과음·배경음), **조이스틱 이미지(신규)**.
- [ ] **EXP② 출제 시퀀스**: 실제 20개 모스 신호 패턴/타이밍/속도, ±5% 판정의 구체 수치(px/ms).
- [ ] **EXP③ 맵 데이터**: 경로 폴리라인 좌표, 막사/태극기 위치, 이탈 ±10% 임계, 차량 속도/가감속 파라미터.
- [ ] **수료증 레이아웃**: 최종 수료증 이미지, input 위치/폰트, 캡처 라이브러리 확정(html2canvas 사용 여부).
- [ ] **유니티 연계 규약**: AR(EXP①) 진입/복귀 방식(URL 파라미터·상태 전달).
- [ ] **도입/연결 자막·내레이션 최종 카피 및 컷 타이밍.**
- [ ] **공통 UX**: 설정값(배경음/효과음 ON/OFF) 저장 범위(세션/localStorage), 스킵 동작 일관화.
